# Case 002 - Suspicious Transaction Pattern

## Case Type

Simulated risk operations case

## Summary

This case reviews a simulated suspicious transaction pattern involving unusual transaction frequency, a new beneficiary, and activity inconsistent with the customer’s expected behavior.

The purpose of this case is to practice transaction monitoring, risk indicator identification, evidence review, and escalation-quality reporting.

## Scenario

A customer account showed a sudden increase in transaction activity within a short period. The account initiated multiple transfers to a new beneficiary that had not previously appeared in the customer’s transaction history.

The activity was inconsistent with the customer’s previous account behavior and may require further review to determine whether the transactions were expected, authorized, or potentially suspicious.

## Observed Activity

| Time | Event | Notes |
|---|---|---|
| 10:05 | Transfer initiated | New beneficiary added |
| 10:12 | Second transfer initiated | Same beneficiary |
| 10:18 | Third transfer initiated | Increased transaction frequency |
| 10:25 | Fourth transfer attempted | Pattern inconsistent with prior activity |

## Key Risk Indicators

- Sudden increase in transaction frequency
- Multiple transfers within a short time period
- New beneficiary added before transaction activity
- Activity inconsistent with previous customer behavior
- Repeated transfers to the same new recipient
- Possible attempt to move funds quickly
- Pattern requiring review before closure

## Evidence Reviewed

- Transaction history
- Beneficiary history
- Customer activity baseline
- Timing and frequency of transfers
- Account change history
- Previous transaction behavior
- Any related alerts or internal notes

## Risk Assessment

The activity presents a potential financial crime or fraud-risk concern. The sudden increase in transaction frequency, combined with transfers to a new beneficiary, creates an unusual pattern compared with the customer’s previous behavior.

The transactions should be reviewed to determine whether they are consistent with the customer profile and whether the activity was authorized. Additional review may be required before the case can be closed.

## Recommended Action

- Review the customer’s previous transaction activity
- Verify whether the new beneficiary is expected
- Check whether the transaction pattern matches the customer profile
- Review any recent account changes or login activity
- Hold or review pending transactions if required by procedure
- Escalate the case if the activity remains unexplained or suspicious

## Escalation Note

Potential suspicious transaction pattern identified. The account showed a sudden increase in transaction activity, including multiple transfers to a newly added beneficiary within a short time period. The activity appears inconsistent with the customer’s previous behavior and should be reviewed to determine whether the transactions were expected, authorized, and consistent with the customer profile.

## Analyst Reflection

This case demonstrates the importance of comparing current activity against the customer’s historical baseline. A single transfer may not be suspicious by itself, but repeated transfers to a new beneficiary within a short period can create a stronger risk pattern.
