# Case 004 - Sanctions False Positive Review

## Case Type

Simulated sanctions screening and risk operations case

## Summary

This case reviews a simulated sanctions screening alert involving a possible name match between a customer and a sanctioned individual.

The purpose of this case is to practice sanctions alert review, identifier comparison, false positive assessment, neutral documentation, and escalation-quality reporting.

## Scenario

A customer generated a sanctions screening alert because the customer’s name appeared similar to a name on a sanctions list.

Initial review showed that the name was similar, but other identifiers such as date of birth, country, and address did not match the sanctioned individual.

The alert requires review to determine whether it is a true match, possible match, or false positive.

## Observed Information

| Data Point | Customer Information | Sanctions List Information | Match Status |
|---|---|---|---|
| Name | Similar name | Similar name | Partial match |
| Date of birth | Different | Different | No match |
| Country | Different | Different | No match |
| Address | Different | Different | No match |
| Entity type | Individual | Individual | Match |

## Key Risk Indicators

- Name similarity with a sanctions list entry
- Initial alert generated by screening system
- Need to compare identifiers before making a decision
- Potential legal or regulatory exposure if the match is confirmed
- Need to determine whether the alert is a true match or false positive

## Evidence Reviewed

- Customer name
- Date of birth
- Country of residence
- Address information
- Sanctions list entry
- Available identifiers linked to the sanctions record
- Internal screening alert details

## Risk Assessment

The alert appears to be a possible false positive based on mismatched identifiers. Although the customer name is similar to the sanctions list entry, the date of birth, country, and address do not match.

A name match alone is not enough to confirm a sanctions match. The available identifiers should be reviewed together before making a risk decision.

Based on the reviewed information, the alert does not currently appear to represent a confirmed sanctions match. However, the case should be escalated if additional identifiers create uncertainty or if internal procedure requires higher-level review.

## Recommended Action

- Document the name similarity and mismatched identifiers
- Confirm that date of birth, country, and address do not match
- Review whether any additional identifiers are available
- Close as a false positive if procedure allows
- Escalate if identifier information is incomplete or conflicting
- Maintain clear documentation of the review decision

## Escalation Note

Possible sanctions screening false positive identified. The customer name is similar to a sanctions list entry; however, key identifiers including date of birth, country, and address do not match the sanctioned individual. A name match alone is not sufficient to confirm a sanctions match. Based on the reviewed identifiers, the alert appears to be a false positive unless additional information suggests otherwise.

## Analyst Reflection

This case demonstrates the importance of identifier-based review in sanctions screening. A similar name can generate an alert, but analysts should avoid treating a name match as a confirmed sanctions match without reviewing supporting identifiers.

The review should remain factual, neutral, and evidence-based. If the identifiers do not match, the alert may be closed as a false positive according to procedure. If uncertainty remains, escalation may be required.
